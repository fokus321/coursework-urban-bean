## 4. Что именно сделано в проекте Urban Bean

### 4.1. Страницы сайта

Сделано 3 страницы:

1. `index.html` — главная страница.
2. `menu.html` — страница меню кофейни.
3. `success.html` — страница подтверждения после отправки формы.

Минимум по заданию — 2 страницы, сделано 3.

### 4.2. Главная страница `index.html`

На главной странице есть:

- шапка сайта;
- навигация;
- hero-блок;
- CTA-кнопки;
- бегущая строка преимуществ;
- блок «О проекте»;
- карточки возможностей сайта;
- видео-секция;
- форма предзаказа;
- подвал с контактами.

### 4.3. Страница меню `menu.html`

На странице меню есть:

- единая шапка и навигация;
- заголовок страницы;
- карточки напитков, завтраков и десертов;
- цены;
- CTA-переход к форме предзаказа;
- общий подвал.

### 4.4. Страница подтверждения `success.html`

После отправки формы пользователь попадает на отдельную страницу подтверждения.

Это закрывает требование: **форма + редирект на отдельную страницу**.

### 4.5. Форма

Форма находится на главной странице.

Поля формы:

- имя;
- телефон;
- email;
- описание заказа.

Реализовано:

- обязательные поля через `required`;
- проверка email через `type="email"`;
- редирект на `success.html` после отправки.

### 4.6. Мультимедиа

В проекте есть:

- SVG-изображения в папке `assets/`;
- hero-иллюстрация кофейни;
- иллюстрация меню;
- встроенное YouTube-видео.

### 4.7. Анимация

Реализованы CSS-анимации:

- бегущая строка преимуществ;
- плавное появление hero-блока;
- hover/focus-анимации кнопок и карточек.

Также добавлен `prefers-reduced-motion`, чтобы уменьшать анимации для пользователей, которым это необходимо.

### 4.8. Адаптивность

Адаптивность реализована через CSS media queries.

Контрольные размеры:

- desktop — широкая версия;
- tablet — средняя ширина;
- mobile — мобильная версия.

На мобильных устройствах сетки перестраиваются в одну колонку, кнопки и навигация становятся удобнее для маленького экрана.

### 4.9. Доступность

В проекте есть:

- `alt` у изображений;
- `aria-label` у навигации;
- skip-link для перехода к основному содержанию;
- корректная структура `h1`, `h2`, `h3`;
- focus-состояния;
- поддержка `prefers-reduced-motion`.

## 9. Что говорить, если спросят на защите

Короткий ответ:

> Я разработал многостраничный сайт городской кофейни Urban Bean на HTML5 и CSS3. В проекте есть главная страница, страница меню и страница подтверждения формы. Сайт адаптирован под desktop, tablet и mobile, содержит форму предзаказа с HTML-валидацией и редиректом, мультимедиа, CSS-анимации и элементы доступности. Код опубликован на GitHub, сайт размещён через GitHub Pages.

Если спросят про форму:

> Форма реализована без JavaScript, средствами HTML. Поля имени, телефона и email обязательные. Email проверяется браузером. После успешной отправки пользователь переходит на отдельную страницу `success.html`.

Если спросят про адаптивность:

> Адаптивность сделана через media queries. На широком экране используются двухколоночные блоки и сетки карточек, на планшете сетка перестраивается, на мобильном всё становится в одну колонку.

Если спросят про доступность:

> Использованы alt-описания, aria-label для навигации, skip-link, корректная иерархия заголовков, focus-состояния и prefers-reduced-motion.

Если спросят про GitHub:

> Репозиторий опубликован на GitHub, сайт развёрнут через GitHub Pages. В истории есть отдельные коммиты и feature-ветка.
